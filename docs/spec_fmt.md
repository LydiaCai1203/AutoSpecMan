# AI SPEC DOCUMENT

## 1. SYSTEM ROLE
定义 AI 的角色与责任。

## 2. TASK
清楚描述你要让 AI 完成的任务。

## 3. RULES
行为规则、限制、禁止事项：
- Rule 1
- Rule 2
- Rule 3

## 4. INPUT FORMAT
定义输入的结构（Markdown、JSON、文本皆可）。

## 5. OUTPUT FORMAT
严格定义输出结构，让 AI 不乱跑版。

## 6. LOGIC / WORKFLOW
流程逻辑或推理步骤（可用项目符号或伪代码）。

## 7. CONSTRAINTS
资源限制、风格限制、长度限制、领域限制等。

## 8. EXAMPLES
提供输入 → 输出示例（AI 最“爱”的部分）。

## 9. FAIL CONDITIONS
明确规定不应出现的行为或错误。

## 10. OUT OF SCOPE
告诉 AI 不要做什么。

## 11. GLOSSARY（可选）
定义术语，减少误解。
