# 开发规范应该包含的内容

## 一、通用规范（前端+后端）

### 1. 项目结构规范
- 目录组织方式
- 文件命名约定
- 模块划分原则

### 2. 代码风格规范
- 缩进、换行、括号使用
- 命名约定（变量、函数、类、常量）
- 注释规范（何时注释、格式）
- 代码长度限制（行数、复杂度）

### 3. 版本控制规范
- Git 工作流（分支策略、合并策略）
- Commit 信息规范（格式、语义化）
- 分支命名规范
- Code Review 流程

### 4. 测试规范
- 测试覆盖率要求
- 单元测试/集成测试/E2E 测试规范
- 测试文件组织方式
- Mock 数据规范

### 5. 文档规范
- README 模板
- API 文档规范
- 代码注释文档规范
- CHANGELOG 规范

### 6. 安全规范
- 依赖管理（版本锁定、安全扫描）
- 敏感信息处理（环境变量、密钥管理）
- 输入验证规范

### 7. 性能规范
- 代码优化原则
- 资源加载优化
- 监控与告警规范

## 二、前端项目特定规范

### 1. UI/UX 规范
- 设计系统/组件库使用规范
- 样式规范（CSS/SCSS/Less 组织方式）
- 主题配置规范
- 响应式设计断点规范

### 2. 组件开发规范
- 组件设计原则（原子化、可复用性）
- Props/PropsTypes 规范
- 组件生命周期使用规范
- 组件文件组织（单文件 vs 多文件）

### 3. 状态管理规范
- 状态管理库使用规范（Redux/Zustand/Pinia 等）
- 状态更新规范
- 状态持久化规范

### 4. 路由规范
- 路由命名规范
- 路由守卫规范
- 懒加载规范

### 5. 构建与打包规范
- 构建工具配置（Webpack/Vite/Rollup）
- 打包优化规范
- 环境变量管理
- 多环境构建策略

### 6. 浏览器兼容性
- 目标浏览器列表
- Polyfill 使用规范
- 降级策略

### 7. 无障碍性（A11y）规范
- ARIA 标签使用
- 键盘导航规范
- 语义化 HTML

### 8. 前端性能优化
- 代码分割规范
- 资源压缩规范
- 缓存策略
- 图片优化规范

## 三、后端项目特定规范

### 1. API 设计规范
- RESTful/GraphQL/gRPC 设计原则
- URL 路径规范
- HTTP 方法使用规范
- 请求/响应格式规范
- 状态码使用规范
- API 版本管理

### 2. 数据库规范
- 数据库设计规范（表命名、字段命名）
- 索引设计规范
- 迁移脚本规范
- ORM 使用规范
- 查询优化规范

### 3. 服务架构规范
- 微服务/单体架构设计原则
- 服务拆分规范
- 服务间通信规范
- 配置管理规范

### 4. 错误处理规范
- 错误码定义规范
- 异常处理策略
- 错误日志记录规范
- 错误响应格式

### 5. 日志规范
- 日志级别使用规范
- 日志格式规范
- 敏感信息过滤
- 日志聚合与分析

### 6. 认证与授权规范
- 认证方式（JWT/OAuth/Token 等）
- 权限管理规范
- 会话管理规范

### 7. 部署与运维规范
- 容器化规范（Docker）
- CI/CD 流程规范
- 环境配置管理（开发/测试/生产）
- 健康检查规范
- 监控与告警指标

### 8. 并发与性能规范
- 并发处理规范
- 缓存使用规范（Redis/Memcached）
- 异步处理规范
- 限流与熔断策略

## 四、技术栈特定规范

- 框架特定规范（React/Vue/Angular、Express/Django/Spring 等）
- 语言特定规范（TypeScript、Python、Java、Go 等）
- 工具链配置（ESLint、Prettier、Black、GoFmt 等）
